# 项目经理常见面试题3：作为项目经理，你是如何进行项目进度管理的？

## 为什么会问这个问题？

进度管理是项目经理的核心职责之一，项目能否按时交付，直接关系到项目成功与否。

面试官希望通过这个问题考察你的：

*   **计划能力**：能否制定合理的项目进度计划。
*   **监控能力**：能否及时跟踪进展，发现偏差。
*   **应变能力**：当进度出现风险时，能否有效调整。
*   **沟通协调**：能否推动团队和干系人共同遵守进度目标。

换句话说，这道题就是在检验你是不是一个真正“能落地执行”的项目经理。

## 千万别这么答

错误示范1（只会列计划）：

> 我会在项目启动时制定好详细的甘特图，然后严格按照计划执行.....

雷区：光说“制定计划”，没有讲到监控和应对措施，容易让面试官觉得你纸上谈兵。

错误示范2（只靠加班）：

> 如果进度滞后，我会要求团队加班赶工，保证按时交付.....

雷区：简单粗暴，暴露了缺乏前置风险预防和调度的能力。

## 建议回答思路

你可以用 **“计划 → 跟踪 → 预警 → 调整”** 四步法来组织回答。

1.  **制定计划**：在项目启动阶段，基于WBS工作分解，把项目拆解到可控任务，明确关键路径和里程碑。
2.  **进度跟踪**：通过项目管理工具（如Jira、TAPD、Project等）定期更新任务完成情况，每周召开站会，检查偏差。
3.  **风险预警**：设立预警机制，比如任务延期超过2天就触发提醒，及时定位问题。
4.  **调整优化**：通过资源调配、优先级调整、范围控制来纠偏，并及时向干系人同步。

## 示例回答（STAR法则）

**S（情境）**：在我之前负责的一个医疗信息化项目中，工期只有4个月，涉及多个子系统集成，进度压力非常大。

**T（任务）**：我的任务是确保项目按期交付，同时保证质量。

**A（行动）**：

1.  我先基于WBS拆解了所有工作包，明确了关键路径和风险点。
2.  每周例会检查里程碑完成情况，同时在Jira里设置了任务延期提醒机制。
3.  当中途遇到接口方交付延迟时，我快速召集协调会，调整了资源优先级，把测试团队提前介入，保证整体节奏不被拖延。
4.  对高风险任务，我单独设立了Buffer，并与干系人同步透明进展。

**R（结果）**：最终项目在计划时间内上线，关键模块提前交付2周，客户验收满意度较高。由于进度管控良好，后续维护阶段的需求变更也更容易被接受。

## 面试官听到后的感受（5点）

1.  **有方法论** —— 不是凭感觉，而是有计划-跟踪-预警-调整的体系。
2.  **能落地** —— 案例具体，工具和机制都能马上照搬。
3.  **控场力强** —— 面对进度风险，能快速协调，避免影响大局。
4.  **结果导向** —— 用“提前交付”“按时上线”等结果增强说服力。
5.  **大局观** —— 不只是盯任务，而是能结合干系人、质量、资源一起考量。

## 实战建议

*   面试前可以梳理1-2个项目进度管理的真实案例，特别是“遇到进度风险 → 如何调整”的场景。
*   如果经验不足，也可以用“假设场景 + 方法论”来回答，表现你的思维逻辑。
*   回答时尽量提到**工具**（Jira、Project、飞书等），这样更显得实际。

## 总结

进度管理不是一张甘特图，而是一套“计划 + 监控 + 预警 + 调整”的闭环。

在面试中，能让面试官感受到你既有方法论，也有实际操作经验，还能应对突发风险，就能为你加分不少。
